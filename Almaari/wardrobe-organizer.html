Wardrobe Organizer

An interactive Wardrobe organizing dashboard where users can upload pictures of clothing items with filtering, create outfits and get tips from Almaari! an AI assistant.


No longer have to worry about finding your clothes in your messy wardrobe. Or spend hours sorting through your clothes to find the perfect outfit. All you have to do is upload your clothes, and the dashboard will do the rest.

Tech Stack

TypeScript Node.js React OpenCV Docker
Live Demo

Features

Clothes Upload

Users can add clothes to their wardrobe by uploading pictures of their clothes.

- Label the type of clothing, color, and style.

- Image background are removed and saved to AWS S3.




Dashboard

Users can see all their clothes in a dashboard, and filter them by type, color, and style.

- Filter by searchbar or sidebar.




Create Outfits

Users can create outfits by picking clothes and saving them to their wardrobe.


Users can also get tips on how to style their clothes from an AI assistant named Almaari!

AWS S3 Image Storage Pipeline

Implemented a secure, scalable upload flow using presigned URLs to avoid routing large files through the backend.

  1. Frontend requests a presigned URL from the backend.
  2. Backend generates the signed URL using the AWS SDK.
  3. Frontend uploads the image directly to S3.
  4. Backend saves an image reference in the database.

Data Architecture & Design

MongoDB with three main collections:

  • User
  • Clothes
  • Outfits

Each clothing item and outfit is stored as an independent document linked to a user ID. This increases modularity and scalability as the platform grows.

Schema Implementation (Mongoose) 

// Clothes Schema
const ClothesSchema = new mongoose.Schema({
  uniqueId: { type: String, required: true, unique: true },
  type: { type: String, required: true, index: true },
  imageSrc: { type: String, required: true },
  favourite: { type: Boolean, default: false },
  createdAt: { type: Date, default: Date.now },
  colour: { type: [String], required: true },
  season: { type: [String], default: [] },
  waterproof: { type: Boolean, default: false },
  slot: { type: String, required: true },
});

// Outfits Schema
const OutfitsSchema = new mongoose.Schema({
  uniqueId: { type: String, required: true, unique: true },
  name: { type: String, default: "" },
  favourite: { type: Boolean, default: false },
  createdAt: { type: Date, default: Date.now },
  colour: { type: [String], required: true },
  season: { type: [String], default: [] },
  waterproof: { type: Boolean, default: false },
  outfit_items: [{ type: mongoose.Schema.Types.ObjectId, ref: "Clothes" }],
});

// User Schema
const usersSchema = new mongoose.Schema({
  auth0Id: { type: String, required: true, unique: true, index: true },
  email: { type: String, required: true },
  clothes: [{ type: mongoose.Schema.Types.ObjectId, ref: "Clothes" }],
  outfits: [{ type: mongoose.Schema.Types.ObjectId, ref: "Outfits" }],
});

Trade-offs & Future Work

  • MongoDB excels for operational workloads (frequent writes, flexible schema), but complex analytics require heavier aggregation.
  • Future enhancement: hybrid system using PostgreSQL for analytics while keeping MongoDB for operations.

Impact & Outcomes

  • Reduced API response times by 25% through query optimization and indexing.
  • Improved data reload speeds using Redis caching.
  • Designed scalable cloud architecture supporting seamless deployment and growth.
